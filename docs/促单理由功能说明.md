# 促单理由功能说明

## 功能概述

在"触达优先级"表中新增了**促单理由**列，自动解释每个客户的优先分是如何计算的，以及为什么要对他进行促单跟进。

## 功能位置

- **Excel输出**: 客户预警输出.xlsx → "触达优先级"工作表 → 第9列"促单理由"
- **HTML仪表盘**: 客户预警仪表盘.html → 促单列表 → "促单理由"列

## 理由组成要素

促单理由由以下6个维度综合生成（按重要性排序）：

### 1. 时机判断 ⏰（最关键）
- **已超流失阈值**: 未购天数明显超过预警阈值，急需挽回
  - 示例：`已超流失阈值(180天>阈值90天)`
- **接近流失临界点**: 即将达到流失阈值，需尽快触达
  - 示例：`接近流失临界点(85天/阈值90天)`
- **即将到达复购期**: 接近正常复购周期，触达时机好
  - 示例：`即将到达复购期(55/60天)`

### 2. 转化潜力 📈
- **高转化潜力**: uplift ≥ 1.3，转化成功概率高
  - 示例：`高转化潜力(uplift 1.6)`
- **中等转化潜力**: 1.0 ≤ uplift < 1.3
  - 示例：`中等转化潜力(uplift 1.2)`

### 3. 利润价值 💰
- **高利润预期**: 估算毛利 ≥ 500元
  - 示例：`高利润预期(¥8000)`
- **中等利润**: 200元 ≤ 估算毛利 < 500元
  - 示例：`中等利润(¥246)`

### 4. 退货风险 ⚠️
- **低退货风险**: 估算退货率 ≤ 10%，订单质量稳定
  - 示例：`低退货风险`
- **需注意退货率**: 估算退货率 ≥ 30%，需留意售后
  - 示例：`需注意退货率(52%)`

### 5. 客户价值层级 👥
- **高价值客户**: 累计消费 ≥ 5000元
  - 示例：`高价值客户(累计¥5251)`
- **中价值客户**: 累计消费 2000-5000元
  - 示例：`中价值客户(¥2858)`

### 6. 订单信心度 ✅
- **老客户**: 3单及以上
  - 示例：`18单老客户`、`7单老客户`
- **待培育**: 2单客户
  - 示例：`2单客户待培育`
- **新客户**: 1单客户
  - 示例：`1单新客户`

## 实际应用示例

### 示例1：高优先级客户
```
木子鱼 (120分)
促单理由: 高利润预期(¥8000) + 低退货风险 + 高价值客户(累计¥5251) + 18单老客户
```
**解读**: 这是一个优质老客户，订单利润高、退货少、消费能力强，值得重点跟进维护。

### 示例2：流失预警客户
```
y (86分)
促单理由: 已超流失阈值(180天>阈值90天) + 高转化潜力(uplift 1.6) + 低退货风险 + 中价值客户(¥2058) + 7单老客户
```
**解读**: 虽然长时间未购买，但历史数据显示转化概率高、退货少，是值得挽回的老客户。

### 示例3：中等价值客户
```
杨艳 (77.71分)
促单理由: 中等利润(¥246) + 低退货风险 + 2单客户待培育
```
**解读**: 2单客户正处于培育期，虽然单笔利润不算高，但退货率低，可以通过小福利唤醒策略促成复购。

## 使用建议

### 对销售人员
1. **优先关注时机**: 有"流失阈值"、"临界点"字样的客户需要尽快触达
2. **评估ROI**: 结合"利润预期"和"转化潜力"判断跟进价值
3. **注意退货率**: 看到"需注意退货率"时，先核实历史退货原因
4. **差异化策略**: 根据"老客户"/"待培育"/"新客户"调整沟通话术

### 对运营管理者
1. **批量筛选**: 在Excel中按"促单理由"关键词筛选特定客户群
2. **培训工具**: 用实际案例培训销售理解优先分逻辑
3. **效果追踪**: 对比不同理由类型的实际转化率，优化算法

## 技术说明

- **生成位置**: `explain_priority_score()` 函数（第599-674行）
- **调用时机**: 在生成action_rows时自动计算（第1297-1307行）
- **输出位置**: Excel第9列、HTML表格对应列
- **更新频率**: 每次运行脚本自动重新生成
